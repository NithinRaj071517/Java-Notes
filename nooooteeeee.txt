-------------------------

		findElement() → Used to locate a single WebElement on the webpage
					    return type WebElement
						returns the first matched elements
						Throws NoSuchElementException
						Single element actions
		findElements() → Used to locate multiple WebElements, returns a list
						return type List of WebElement
						Returns all matched elements

Waits:
------
	Definition:
		Waits in Selenium are used to synchronize the WebDriver with the application, ensuring that elements are loaded or visible before interaction, instead of failing immediately.
		
	Need:
		We need waits in automation testing because modern web applications are dynamic and elements may load due to AJAX calls, page refreshes, or delayed responses. Without waits, Selenium may try to interact with elements that are not yet available.
	Types:
		1. Implicit Wait
			Global wait applied to all elements
			WebDriver waits up to the specified time before throwing NoSuchElementException
			Defined once and applied throughout the driver session
			
		2. Explicit wait
			Applied to specific element Wiats for a  specific condition.
			Uses WebDriverWait with ExpectedConditions
				WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(20));
				wait.until(ExpectedConditions.visibilityOf(element));
			
		3. Fluent wait
			Similar to explicit wait with polling frequency and Exception ignoring
			
			
	Although Selenium allows using both waits together and it does not throw any exception, it is not recommended because implicit wait gets added to explicit wait internally, resulting in unpredictable and increased execution time.Hence, in real-time automation frameworks, we usually disable implicit wait and rely on explicit waits for better control.		
			
StaleElementReferenceException:
------------------------------			
	Definition:
		StaleElementReferenceException occurs when a WebElement reference becomes invalid, meaning the element is no longer attached to the current DOM.			
	Causes:
		1. Page refresh
		2. DOM updated due to AJAX
		3. Element removed or recreated
		3. Element removed or recreated
		4. Switching frames or windows
		
	Solutions:
		1. Relocate the element
		2. Use explicit wait 
		3. Try catch retry logic
		
	Examples:
		After clicking "Save", page refreshes → element becomes stale → need to re-locate before next action.

ArrayList and LinkedList:
------------------------		

	Definition:
		ArrayList and LinkedList are implementation classes of the List interface used to store ordered and dynamic collections of elements.

	ArrayList:
		1. It is an dynamic array
		2. Its access time is fast
		3. Insertion/Deletion is slow
		4. Uses a resizable array
		5. When capacity is full → creates a new array (approx 1.5x size) and copies elements
		6. It follows the insertion order and allows duplicates
	 

	LinkedList:
		1. It is doubly linked list(node based)
		2. Its access time is slow 
		3. Insertion/Deletion is fast 
		4. Uses nodes (each node has data + next + previous reference)
		5. No shifting required during insert/delete
		6. It follows the insertion order and allows duplicates


HashSet & HashMap:
-----------------
	Definition:
		HashSet is  implementation classes of the Set interface , stores unique elements
		HashMap is  implementation classes of the Map interface , stores key-value pairs

	HashSet:
		1. Stores only value
		2. Insertion order not maintained 
		3. Duplicates are not allowed 
		4. Allows to add one null value
		5. Uses HashMap internally
	 

	HashMap:
		1. Stores  key value pair representation 
		2. Insertion order not maintained 
		3. Duplicates Keys are not allowed but duplicate values allowed 
		4. Allows one null value in key & multiple null values
		5. HashMap works based on hashcode() and equals() for fast retrieval

	HashSet is used in window handling for unique window ID
	and HashMap used in test data assigning like user name & password

Memory Allocation:
-------------------
	In Java, array memory is managed using both Stack and Heap
		Stack memory --> Heap stores reference 
		Heap memory --> Heap stores elements
	
	int[] arr = new int[3];
		arr --> reference varaible --> stack memory
		int[3] --> creates 3 elements in array --> heap memory
	  
	Heap allocates continuous memory location , so accessing is faster .
	Supports index based access

Select Dropdown:
---------------
	Definition:
		Select class is used to handle dropdown elements that are created using the <select> HTML tag.
		The Select class supports dropdown handling only when the tag name is <select>
		If the dropdown is not built using <select>, then we handle them using normal click actions or Actions class.
		
	Select options using:
		selectByVisibleText() – based on displayed text
		selectByValue() – based on value attribute
		selectByIndex() – based on index position
		isSelected() --> returns true if dropdown value selected 
		getOptions()--> method of the Select class to retrieve all available options from a dropdown and return List of WebElement
		getAllSelectedOptions()->  method gives all selected options
		
	Multiple Select:	
		In selenium, we can check whether a dropdown supports multiple selection using the isMultiple() method of the Select class.	
		If the <select> tag contains the multiple attribute, then the dropdown allows selecting multiple values.
		Deselect operations are supported only for multi-select dropdowns, not for single-select dropdowns.
			isMultiple() method --> Used to check it supports multi-select
			sel.deselectByIndex(1);
			sel.deselectByValue("IN");
			sel.deselectAll();

Alerts:
-------
	Definition:
		Alerts in selenium is a Javascript popup used by  the application to display information, confirmation and 	or request input from user.
		Javascript alerts are  not part of the DOM ,so it cannot be inspectable .
		If an alerts appears and  are not handled , selenium execution blocks and no further actions can be performed.
		Selenium handles alerts using the Alert interface.
		
	Types:
		Simple Alert – Information message with OK button
		Confirmation Alert – OK and Cancel buttons
		Prompt Alert – Input box with OK and Cancel buttons	
		
	Examples:
		1. You have an alert and you ignored it  --> UnhandledAlertException --> This happens because alerts block the UI thread until they are accepted or dismissed.
		2. You dont have an alert and you handled it  -> NoAlertPresentException
		
		
Locators:
--------
	Definition:
		Locator is used to identify and locate web elements on a web page so that Selenium can perform actions like click, sendKeys, or getText.
		Since web applications are built using HTML, locators help Selenium uniquely identify elements from the DOM.
		
	Types:
		1. Basic Locators 
				Id
				Name
				ClassName
				TagName
		2. Advanced Locators
				Link text
				Partial link text
				CSS selector
				Xpath
	
	Most preferred locator in Selenium:
		1.By.id because:
			IDs are unique in the DOM
			They are fastest for element identification
			They make scripts more stable and readable
		2. If Id is not available next preferred
			Name
			css selector
			Relative Xpath
			
	Less preferred locator in Selenium:
		1. By.classname:
			Same class can be used for multiple elements, which may lead to incorrect element identification if not handled properly.
		2. Absolute xpath:
			It is tightly coupled to the DOM structure and breaks easily when UI changes.
	
	CSS vs Xpath:
		1. CSS:
			CSS Selector uses CSS syntax to locate elements based on attributes such as id, class, tag name, and relationships.
			Syntax simpler
			DOM traversal forward only
			Performance is faster
			
		2. Xpath:
			XPath uses XML path expressions to locate elements by traversing the DOM structure.
			Syntax more powerful
			DOM traversal both backward & forward 
			Performance is slightly slower
			
		Preferred CSS because it is faster , cleaner and easy to maintain. 
		However , Xpath is unavoidable when 
			Locating elements using text
			Traversing from child to parent
			Handling complex dynamic elements
			
			
String:
------
	Definition:
		String in Java 	is an object that represents sequence of characters, implemented using String class from java.lang package.
		Strings are immutable, meaning once created, their value cannot be changed. If any modification is performed, a new object is created instead of modifying the existing one.
		
	Memory allocation:
		String literals are stored in the String Constant Pool (SCP) inside the heap. If the same value already exists, JVM reuses the reference to optimize memory.
		
		String objects (new keyword) are always created in the heap memory, even if the value already exists.
			
			String s1 = "Test";
			String s2 = "Test";  // same reference from SCP
			String s3 = new String("Test"); // new object in heap
			
	Reason for immutability:
		Security (important for URLs, DB connections)
		Caching (String pool reuse)
		Thread safety (no synchronization required)		
		
		

String vs String Buffer vs String Builder:
-----------------------------------------
	Definition:
		String, StringBuffer, and StringBuilder are classes used to represent sequences of characters in Java, but they differ in mutability, thread safety, and performance.
		
	
	Difference:
		1. String:
			Immutable
			Thread safe
			Stored in Sting Constant Pool
			
		2. String Buffer:
			Mutable
			Thread safe
			Slower due to synchronization overhead.
		
		3. String Builder:
			Mutable
			Not Thread safe
			Faster due to no synchronization .

Access Modifiers:
----------------
	Definition:
		Access Modifiers in Java are used to control the visibility and accessibility of classes, methods, variables, and constructors.
		
	Types:
		1. Public - Accessible from anywhere
		2. Private - Accessible only within the same class
		3. Protected - Within same package 	
					Outside package only through inheritance 
		4. Package(Default)- Accessible only within the same package
							No keyword used
		
	Uses :
		private → WebElements / variables (encapsulation)
		public → reusable methods (e.g., login(), sendRequest())
		protected → base class methods for inheritance
		default → internal utility classes within package
		
	Can we make a class as private or protected in Java:
		In Java, top-level classes cannot be declared as private or protected but Inner Classes CAN be private or protected.
		A top-level class should be accessible from other classes/package
			If we make it:
					private → not accessible outside → useless
					protected → not valid at top-level (Java doesn’t allow it)
					
					
Constructor:
------------
	Definition:
		A constructor is a special method that is automatically invoked when object is created for class.
		It primary purpose is to initialize the instance variables.
		
	Characteristics:
		Same name as class
		No return type (not even void)
		Automatically called during object creation
		Can be overloaded
	
	Types:
		1. Default Constructor  -> Provided by JVM if no constructor is defined
		2. Parameterized Constructor  -> Used to initialize variables with custom values
		
	Examples:
		Used to initialize WebDriver, config, DB connection
		
	Constructor vs Method:
		
		Constructor:
			Special method used to initialize object
			Same name as class
			No return type
			Automatically called when object is created
			Can be overloaded

		Method:
			Used to define business logic / operations
			Can have any name
			Must have return type (or void)
			Must be explicitly called
			Can be overloaded
			
	Constructor Overloading:
		Constructor overloading in Java means defining multiple constructors in the same class with different parameter lists (type, number, or order of parameters).		
		
		Example:
			Used in cross browser execution concept in selenium
